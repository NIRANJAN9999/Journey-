01

The curiosity

It started with remote control cars from the local Mahankaali and Shivaratri fairs — and the need to know what was inside them. No screwdriver, so a sharpener blade and a pair of scissors did the job. Not everything went back together working, and there were consequences for that at home. The curiosity never really left.

02

Learning by breaking

In 7th grade, I wired my new cycle with hidden lights that only switched on when I braked — using the brake pads themselves as the switch. A friend later showed me how to spin an old motor to generate current and light an LED, jugaad-rigged to the wheel. When it burned out, I opened it up and fixed it with copper wire. I also learned, the hard way, that a 12V LED does not appreciate an AC socket.

03

Finding the path

A diploma in mechanical engineering came first — gears, engines, CAD basics, real machining in the labs. Then a deliberate switch to a B.Tech in computer science, chasing the coding and electronics itch that never left.

04

Building for real

Got a laptop, installed the Arduino IDE, bought a few microcontrollers, and started building from YouTube tutorials — code that sometimes worked, sometimes didn't, slowly learning why. Taught myself phone repair the same way. That path led to ESP32, home automation, and a self-hosted server running a multi-agent AI system today.

I never stopped opening things up.

Home automation with ESP32

3D printing & resin casting

Started with an FDM printer a year ago. Now experimenting with resin and silicone molds to get past visible layer lines.

Wood-and-print table lamp

A table lamp with a wood base and a 3D-printed shade — my first real attempt at turning a build into a small product.
